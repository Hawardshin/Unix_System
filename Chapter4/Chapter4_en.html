Chapter4 Vim manunal

Type of Unix Editor

1. Row-by-row editor

  • - ed: Used early in Unix → rarely used now
  • -ex: Use in connection with vi rather than alone

2. Screen unit editor

  • vim: Screen editor similar to vi (solaris runs vim when using vi)
  • Emacs: Powerful function but requires installation

Mode

1. Mode type (Vim)

  • Key entered in command mode is considered a command
  • Key entered in input mode is considered data
  • There is a special key to change mode (e.g. Esc)

2. Non-mode type (Korean, Word)

  • All keys entered are considered data
  • The command consists of a combination of special and regular keys

How to use vim

  1. Call files from disk.
  2. Modify the file in the editor.(Save in Temporary Buffer)
  3. When you save the file, it is saved to disk.

Characteristics

  • If you modify the file contents incorrectly, you can ignore the file modifications and leave. (Ignores buffer content)
  • There is a job cancellation function.

Change operation mode

1. Command mode → Input mode

i Enter from the current cursor position I Enter from the beginning of the current cursor line (shift + i)
a Enter the current cursor from the next column A Enter from the end of the current cursor line (shift + a)
o Enter the next line of the current cursor O Enter the previous line of the current cursor (shfit + o)
s Clear one letter of the current cursor position and enter S Erase one line of the current cursor and enter (shift + s)

2. Last row mode

  • Mode that searches, saves, and moves rows
    • Use
  • :, /, ? to move mainly
    • Use
  • w, qq!, etc. (ZZ==wq)

3. Return to command mode from another mode: ESC

vim Move cursor

  • Up, down, left, right: k,j,h,l
  • Going to the beginning of the current row: ^ or O
  • Move to the end of the current row: $
  • Move the cursor to the beginning of the previous row: -
  • Move the cursor to the beginning of the next line: +
  • Cursor at the top of the screen, middle bottom: H, M, L
  • Cursor is the first letter of the next word, the first letter of the previous word, the last letter of the next word: w,b,e

Moving screen

  • Up/down on one screen: ctrl+b, ctrl+f
  • Top/Bottom of the screen: ctrl +u, ctrl +d

Go to the specified row

  • :Row number: Go to the specified row
  • :$ Move to last row

Modify and delete

  • r: Modify cursor position characters to other characters
  • cw, #cw: Modify the cursor position to the end of the current word with the new input, enter Esc, and the modification is finished. (# can be used like 3cw words to modify)
  • s, #s: Modify the character where the cursor is located to the content you enter until you enter the ESC key (# is the number of characters to modify)
  • cc: Modify all the contents of the row where the cursor is located
  • C: Modify cursor position to end of row
  • x, #x: Delete the character where the cursor is located (#Delete the character)
  • dd #dd, : Delete row where cursor is located
  • u: Undo

How to cancel undo again

  • `.` :redo to write the command you just did again
  • Change 'r' characters Replace 'ra' characters with 'a'.

Edit Function

  • yy #yy: Copy the row where the cursor is located (copy cursor position 3 rows if 3yy)
  • p: Attach to the bottom of the row where the cursor is located.
  • dd #dd: Cut the row where the cursor is located. (# specifies the number of rows to cut)

Buffer

  • vim saves work to buffer → undoable
  • Copy, use for cut

Buffer Type

  • Unnamed buffer: Unnamed buffer
  • Named buffers: named buffer
  • Numbered buffers: numbered buffers

Usage example

  • "a3yy → Save 3 lines down from the current row to a buffer
  • "ap → aBuffer Addition

Copy and Cut in Last Row Mode

Range

  • 1, $, or %: Specify line 1 to line last
  • 1,. : Specify from row 1 to row with cursor
  • , $: Specify the row with the cursor to the last row
  • ,-3 : Specify from current row to forward 3rd row
  • 10,20: Specify lines 10 to 20

Copy/Cut (Not on the test)

  • :#y: Copy the row specified by # (:10y: Copy 10 rows)
  • :y: Copy rows specified in the range (:10,20y: Copy rows 10 to 20)
  • Delete row specified by:#d:# (:10d:Delete row 10)
  • :d: Copy rows specified in the range (:10,20d: delete rows 10 to 20)]
  • :pu: Attach the contents of the buffer after the current row.
  • #pu: Attach the contents of the buffer after the row specified by #.

Change Search

  • /String: Search string down
  • ? String: Search string up
  • n: Find the next string in the original direction
  • N: Find the next string in the reverse direction
  • :s/string1/string2: Replace the first string1 of the row where the cursor is located with string2.
  • :s/string1/string2/g: switch across files
  • :s/string1/string2: Replace only the first string of each row of all rows in the range (all strings in all rows if g is attached)

Other commands

  • ':r File name: Load existing file.`
  • ':e File name: Switch to the specified file.`
    • Press
  • `ctrl' to align the screen. (Use when the screen is cracked)
  • :r File name: Read the specified file and insert it into the current cursor location
  • :e File name: Switch to the specified file (Save and run the existing file)
  • :n Go to next file

Execute shell command in vim

  • :! Shell command: Stop vim for a while and execute shell command (Enter to return)
  • :sh:Exit vim for a while and execute shell command (exit to return)

  • :set:output all vim variables set to set
  • :set all: Output all vim variables and current values
