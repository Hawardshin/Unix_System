유닉스 편집기의 종류

1. 행 단위 편집기

  • - ed : 유닉스 초기에 사용 → 지금은 거의 사용하지 않음
  • - ex : 단독으로 사용하기 보다는 vi와 연결하여 사용

2. 화면 단위 편집기

  • vim : vi와 유사한 화면편집기 (솔라리스는 vi를 사용시 vim실행)
  • Emacs : 막강한 기능 있으나 설치 필요

모드

1. 모드형(Vim)

  • 명령모드에서 입력한 키는 명령으로 간주
  • 입력모드에서 입력한 키는 데이터로 간주
  • 모드를 바꾸기 위한 특수 키가 있음 (예: Esc)

2. 비모드형(한글, 워드)

  • 입력한 모든 키는 데이터로 간주
  • 명령은 특수키와 일반키의 조합으로 구성

vim의 사용 방법

  1. 디스크에서 파일을 호출한다.
  2. 편집기에서 파일을 수정한다.(임시 버퍼에 저장)
  3. 파일을 저장하면 디스크에 저장이 된다.

특징

  • 파일 내용을 잘못 수정한 경우 파일 수정 내용을 무시하고 나갈 수 있다. (버퍼의 내용 무시)
  • 작업 취소 기능이 있다.

동작모드 바꾸기

1. 명령모드 → 입력모드

i 현재 커서의 위치부터 입력 I 현재 커서 줄의 맨 앞에서부터 입력 ( shift + i )
a 현재 커서의 위치 다음 칸부터 입력 A 현재 커서 줄의 맨 마지막부터 입력 ( shift + a)
o 현재 커서의 다음 줄에 입력 O 현재 커서의 이전 줄에 입력 ( shfit + o )
s 현재 커서 위치의 한 글자를 지우고 입력 S 현재 커서의 한 줄을 지우고 입력 ( shift + s )

2. 마지막행 모드

  • 검색, 저장, 행이동을 하는 모드
  • :, /, ? 를 사용해서 주로 이동
  • w, q q!등을 사용(ZZ ==wq)

3. 다른모드에서 명령모드로 돌아가기 : ESC

vim 커서이동

  • 상하좌우 : k,j,h,l
  • 현재 행의 처음으로 이동 : ^ 또는 O
  • 현재 행의 마지막으로 이동 : $
  • 커서를 이전행의 처음으로 이동 : -
  • 커서를 다음행의 처음으로 이동 : +
  • 커서를 화면 맨위, 중간 아래 : H, M, L
  • 커서를 다음단어의 첫글자, 앞단어의 첫글자 다음단어의 마지막 글자 : w,b,e

화면이동

  • 한화면 위/아래 : ctrl + b , ctrl + f
  • 반화면 위/아래 : ctrl + u, ctrl + d

지정한 행으로 이동

  • :행번호 : 지정한 행으로 이동
  • :$ 마지막 행으로 이동

수정 및 삭제

  • r : 커서 위치 문자를 다른문자로 수정
  • cw, #cw : 커서의 위치부터 현재 단어의 끝까지를 새로 입력한 내용으로 수정, Esc를 입력하면 수정이 종료됨.(#은 수정할 단어 수 3cw이렇게 사용가능)
  • s, #s : 커서가 위치한 문자를 ESC키 입력할 때까지 입력한 내용으로 수정(#에는 수정할 문자수)
  • cc : 커서가 위치한 행의 내용을 모두 수정
  • C : 커서의 위치부터 행의 끝까지 수정
  • x, #x : 커서가 위치한 문자 삭제(#개 문자 삭제)
  • dd #dd, : 커서가 위치한 행 삭제
  • u : 실행취소

undo 를 또 취소 하는법

  • `.` : redo 방금 한 명령을 한번 더 쓰는것
  • `r` 문자 바꾸기 r a문자를 a로 바꾼다.

편집기능

  • yy #yy : 커서가 위치한 행을 복사 (3yy하면 커서위치 3행 복사)
  • p : 커서 위치한 행의 아래 쪽에 붙인다.
  • dd #dd : 커서가 위치한 행을 잘라둔다. (#에는 잘라둘 행 수 지정)

버퍼

  • vim은 작업 내용을 버퍼에 저장 → 실행 취소 가능
  • 복사하기, 잘라내기에 사용

버퍼 종류

  • Unnamed buffer : 이름 없는 버퍼
  • Named buffers : 이름이 있는 버퍼
  • Numbered buffers : 번호가 있는 버퍼

사용 예

  • “a3yy → 현재 행부터 아래로 3줄을 a버퍼에 저장
  • “ap → a버퍼의 내용을 붙이기

마지막 행 모드에서 복사하기와 잘라내기

범위 지정하기

  • 1, $또는 % : 1행부터 마지막 행까지 지정
  • 1 ,. : 1행부터 커서가 있는 행까지 지정
  • ., $ : 커서가 있는 행부터 마지막 행까지 지정
  • ,-3 : 현재 행부터 앞으로 3번째 행까지 지정
  • 10,20 : 10행부터 20행까지 지정

복사하기 /잘라내기 (시험에 안나옴)

  • :#y : #으로 지정한 행 복사 (:10y : 10행 복사)
  • :<범위>y : 범위로 지정한 행 복사 (:10,20y : 10~20행까지 복사)
  • :#d : #으로 지정한 행 삭제 (:10d :10행 삭제)
  • :<범위>d : 범위로 지정한 행 복사 (:10,20d : 10~20행까지 삭제)]
  • : pu : 현재행 다음에 버퍼의 내용을 붙힌다.
  • #pu : #으로 지정한 행 다음에 버퍼의 내용 붙힌다.

검색하기 바꾸기

  • /문자열 : 문자열을 아래방향으로 검색
  • ? 문자열 : 문자열을 위 방향으로 검색
  • n : 원래 찾는 방향으로 다음 문자열 찾기
  • N : 역방향으로 다음 문자열 찾기
  • :s/문자열1/문자열2 : 커서가 위치한 행의 첫번째 문자열1을 문자열2로 바꾼다.
  • :s/문자열1/문자열2/g : 파일 전체에서 바뀍
  • :<범위>s/문자열1/문자열2 : 범위 안의 모든행의 각 행의 첫번째 문자열만 바꾸기(g를 붙히면 모든행에 모든문자열)

기타 명령어

  • `:r 파일명 : 기존의 파일을 불러온다.`
  • `:e 파일명 : 지정된 파일로 전환한다.`
  • `ctrl l` 을 누르면 화면정렬이 된다.(화면이 꺠질 때 사용)
  • :r파일명 : 지정한 파일 읽어서 현재 커서 위치로 삽입
  • :e 파일명 :지정한 파일로 전환 (기존 파일 저장후 실행)
  • :n 다음파일로 이동

vim에서 쉘명령 실행

  • :! 쉘명령 : vim을 잠시 중단하고 쉘명령 실행 (돌아오려면 엔터)
  • :sh : vim을 잠시 빠져나가서 쉘명령 실행 (돌아오려면 exit)

vim 환경설정

  • :set nu(nonu) : 각 행 번호표시(감추기)
  • :set : set으로 설정한 모든 vim 변수 출력
  • :set all : 모든 vim변수와 현재 값 출력
