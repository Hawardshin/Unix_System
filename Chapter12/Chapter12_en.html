Chapter 12.Understanding Unix Shell

Shell

What is shell: Programs that provide a user interface

Shell function

  • Command interpreter function
    • Interpret and forward commands between the user and the kernel.
    • Interpret commands that you enter or read from a file and run the appropriate program
  • Programming function
    • The shell has its own programming function, so it can be programmed
    • You can use multiple commands to produce repetitive tasks into a single program.
  • User Configuration Features
    • Setting up the user experience using the initialization file function
    • Each user can set the characteristics of the environment to the initialization file, including setting the path to the command, setting default permissions for new files, and setting various environmental variables.
    • This initialization file runs when you log in to set up your initial environment

Type of Shell

Bone Shell (sh)

    Li is the first shell.
  • Many shell scripts that perform system management tasks are based on this shell.
  • It does not provide user convenience functions such as history function, alias function, and task control.
  • In most Unix systems, the bonshell (/usr/bin/sh) is replaced by a cornshell on Solaris 11

C shell(csh)

  • Expanding the features of Bonshell, including user convenience features such as Alias and History
  • The syntax for writing shell scripts is the same as the C language, so it is called the C shell
  • alias clear (alias command) is different.)
  • ! 23, !l : Execute 23rd command!l : Execute command starting with l.
  • General user: %, root user: #
  • setenv [environment variable name value]
  • set [environmental variable name = value]
  • . login : Run only once when logging in, environment variables, terminal settings
  • .cshrc: When creating a subshell when logging in, shell variables, aliases, and history settings

Conshell (ksh)

  • Maintains compatibility with Bone Shell
  • It has the advantage of fast processing speed while providing all the features of the C shell such as history function and alias function.
  • General user: $, root user: #
  • Set environment and shell variables: Same as bash
  • Save it to the initialization file.kshrc and it will be aliased every time you log in
  • History setting (HZ=200): Meaning that 200 commands can be saved Default value is 100
  • Save recent HZ commands to a history file (~/.sh_history) when logging out
  • history command: Number 16 recently used commands together. (r rerun command)
  • Features: Direction key function, tab auto complete

System Initialization File

  • /etc/profile: This file will be initialized by all users on the system.
  • ~/.profile : Provided by default to all users using the cornshell, saving environment variable settings such as configuration files, prompts, and command paths that vary from system user to system user
  • ~/.kshrc: A configuration file from a cornshell, also runs on a child shell, alias settings, configuration settings using set

Bash Shell

Includes convenient features of C shell and console while maintaining compatibility with this shell.

It is an open software that follows GNU and acts as the default shell for Linux, also known as the Linux shell

stty terminal function setting

  • Set terminal option function
  • If you do not specify an option and mode, output the current setting value
  • Option -a: Outputs terminal-wide option values
  • Mode
    • erase: Specify delete key
    • rows: Specify the number of rows in the terminal
    • columns: Specify the number of columns in the terminal
