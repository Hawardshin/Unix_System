파일 아카이브 : tar

tar(tape archive)는 디렉토리가 여러 가지로 연결되어 있는 것을 하나로 묶은 것입니다. 압축은 되어 있지 않습니다.

옵션

  • tar cvf . my.tar : 현재 디렉토리 아래 모든 폴더를 압축 (아카이브 생성)
  • tar xvf my.tar : 압축 해제
  • tar tvf my.tar : 목록을 볼 수 있습니다.
  • uvf : 업데이트
  • rvf : 파일 추가하기

gzip

gzip은 확장자가 .gz인 압축파일을 생성합니다. 압축률이 더 높습니다.

옵션

  • -d 옵션: 압축 해제 (gunzip과 동일)
  • -l : 압축된 파일의 내용 출력
  • -r : 현재 디렉토리부터 하위 디렉토리까지 모두 압축
  • -t : 압축 파일 검사
  • -v : 압축 정보 화면에 출력
  • -g : 최대한 압축

Zip

Zip은 .gz인 압축파일을 생성합니다.

옵션

  • -v : 압축 정보 화면에 출력
  • -r : 디렉토리 압축
  • -u : 수정되거나 추가된 파일만 출력

bzip2

bzip2는 gzip, zip에 비해 압축률은 높고 속도는 약간 느립니다. .bz2 파일을 생성합니다.

옵션

  • -d : 파일 압축을 해제
  • -l : 압축된 파일의 내용을 보여줌.

    • Zip

    • zip [option] filename
    • .gz인 압축파일 생성
    • unzip으로 압축해제

    option

    • -v : 압축 정보 화면에 출력
    • -r : 디렉토리 압축
    • -u : 수정되거나 추가된 파일만 출력

    bzip2

    • 오픈 소스 압축명령
    • gzip, zip에 비해 압축률은 높고 속도는 약간 느리다.
    • .bz2 파일 생성

    옵션

    • -d : 파일 압축을 해제
    • -l : 압축된 파일의 내용을 보여줌.
    • -t : 압축 파일 검사
    • -v : 압축 정보를 화면에 출력
    • --best : 최대한 압축

    다른 명령들

    • jar : Java Archive Tool 여러 파일을 하나의 jar파일로 묶어서 압축, 옵션없이 사용하면 사용법 출력한다!
    • compress (compress [옵션] 파일명) : 파일을 압축한다. 확장자가 .z인 파일을 생성한다.
    • uncompress : 압축 해제
    • zcat : zcat [파일명], .z파일 내용을 화면에 출력.
    • gzcat : 확장자가 .gz인 파일 내용 화면에 출력.
    • zip : 확장자가 .gz인 압축파일 생성
    • unzip : 확장자가 .gz인 압축 파일 생성
