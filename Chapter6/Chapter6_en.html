Chapter6 file Access rights

Features of multi-user systems

  • Multiple people use one system
  • Requires security so that no one can read, modify, or delete my files
  • Using files only for permissions by granting access to files with the ability to protect your files from unauthorized access by other users

File Properties

  • File properties are information that Unix stores to manage files, and you can check the file properties with the ls–l command.
  • File type (-: General file, d: directory)
  • File access rights
  • Number of hard links
  • User name of file owner
  • Name of the group to which the file belongs
  • File size
  • Last modified date and time of file
  • Related commands: file file name → File type identification command

File access rights

  • Access to read, write, and execute files
  • It is the most basic security function of a multi-user system.
  • Access rights are granted differently for each user category.

Basic Permissions

  • [Owner][Group User][Other Users]
  • [rwx][rwx][rwx]
  • Read(r), Write(w), Execute(x)
  • Meaning differs depending on the type of file
  • Read: File can be read or copied. (For directories, you can view the directory list with the ls command.)
  • Writing: You can modify, move, and delete files. (You must have write permissions in the directory.) (write permissions in the directory: You can create or delete files.)
  • Run: You can run the file in the case of shell scripts or executable. (In the case of a directory, a CD command is possible, and files can be moved or copied to the directory.)
  • Authority granted when a file is created by default in Unix: general files that cannot be executed (6666), general files that can be executed (777), directory (777)

Change permissions

  • chmod [option] mode filename
  • Use to change the permissions of your own files (-R: Change all subdirectories)
  • User: u (owner), g (group), o (other user), a (all users) (u+g+o)
  • Operators: +(granting permission), -(removing permission), =(designating permission to specific users)
  • Authority symbols: r (read), w (write), x (execute)
  • Examples of use: u+w, u-x, o=rwx, go-wx
  • Change permissions to numeric mode: use octal.

umask [mask value] command

  • Use to output or change default access rights
  • Mask Value
    • Specifying permissions not to grant when creating files or directories
    • If you specify this value, specify default access using the specified mask
    • If not specified, output the currently set mask value
    • Default Permission XOR Mask
  • Maximum privileges: General file (rw-rw-rw-) (666), Directory (rwxrwxrwx) (777)
  • Basic license limit and mask value and XOR mask.
