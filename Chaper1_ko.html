UNIX 1강

유닉스 개요

유닉스의 역사

  • 1969. AT&T 벨 연구소에서 개발 (개발자 : 켄 톰슨, 데니스 리치)
  • 현재 UNIX는 “The Open Group”의 등록 상표이므로, 각 제조사는 각자 제품명을 붙여 사용

Linux란?

  • 유닉스와 호환 가능한 유닉스 계열 운영체제
  • 개발 초기부터 공개 S/W
  • 1991. 리누스 토발즈가 개발
  • 현재 토발즈가 리눅스 커널의 개발을 중재

유닉스의 특징

  • 대화형 시스템
  • 다중 사용자 시스템 / 다중 작업 시스템
  • 높은 이식성과 확장성
  • 계층적 트리 파일 시스템
  • 다양한 부가 기능 제공

유닉스의 구조

  1. 커널
    • 운영체제의 핵심
    • 컴퓨터 자원 관리(프로세스, 메모리, 파일시스템, 장치)
  2. 셸
    • 사용자 인터페이스 제공
    • 사용자의 명령을 입력 받아 처리 한 후 결과를 출력
  3. 유틸리티
    • 유닉스 시스템이란 유닉스가 동작하고 있는 컴퓨터를 의미한다.
    • 유닉스 시스템에 접속하기 위해 필요한 것 : 단말기, 사용자 계정
    • 단말기 : Putty, 하이퍼 터미널, 텔넷, SSH(텔넷의 보안문제 해결)
    • 사용자 계정 : 사용자 ID, 비밀번호지정(시스템 관리자에게 요청하여 등록가능)

로그인 하기 실습

로그인 하기

  • 시스템 사용을 허가 받고 자원을 할당 받는 과정
  • ID와 비밀번호 입력

[실습 1-1] ssh를 이용해 유닉스에 접속하기

  1. cmd 열기
  2. 사용자 계정의 ID @ 서버의 IP 주소 입력
  3. ssh인증키 생성
  4. 유닉스 시스템 로그인

집에서 연결이 안되는 이유

  • 인터넷 주소는 public ip(공인 ip) 와 private ip(사설 ip)가 있는데 우리의 10.10.54.75 주소는 사설 IP이기 때문이다. (사설 IP는 주소 중복이 가능)
  • 사설 IP에대한 정리는 제 블로그에 해두었습니다.

telnet이 안된다면

  • 제어팟/모든제어판 항목/프로그램 및 기능/window 기능 켜기/끄기→ 텔넷 클라이언트를 체크해주면 된다.
  • 문제 해결 링크

Command Line Operation

명령행 편집

  • 문자 지우기 : 백스페이스
  • 단어지우기 : Ctrl + W
  • 문장 지우기 : Ctrl + U
  • 잠깐 중단 : Ctrl +s
  • 다시 시작 : Ctrl + q

명령의 구조

명령 [옵션] [인자]
  • 명령 : 유닉스에 작업을 지시하는 프로그램 이름으로 쉘 빌트인 명령과 유틸리티로 구분된다.
  • 옵션 : 명령의 세부기능을 선택한다. 보통 “-”로 시작하고 숫자, 대문자, 소문자를 사용한다.
  • 인자 : 명령으로 전달되는 값으로 보통 파일명이나 디렉토리 명이다.
  • 유닉스 명령은 대소문자를 구분한다.
  • 맨 앞에 있는 $의 의미는 프롬포트이다.

기초 명령

  • banner : 화면에 큰 문자를 출력 -사용법 : `banner [원하는 문자]`
  • date : 시간과 날짜를 출력
  • clear : 화면을 지움 (in ms window cls)
  • man : 명령어 설명 출력
  • passwd : 비밀번호 변경(8자리 이상 숫자나 특수문자 포함하여 지정) (관리자도 리셋만 가능)
  • logout, Exit, Ctrl + D 명령 : 쉘을 종료, 로그아웃하는 명령어
  • ping 10.10.54.75 : 대상 ip를 가진 컴퓨터를 향해 일정 크기의 패킷을 보내고 대상 컴퓨터가 이에 응답을 보내는것을 수신해서 네트워크 동작 여부 및 상태를 파악하는 명령어 (`ping [google.co.kr](http://google.co.kr)` 도 가능)
  • ipconfig 명령 : DNS, IP Address,Subnet Mask, Gateway의 주소를 볼 수 있는 명령 (리눅스 계열에서는 ifconfig로 이름이 다르다)
  • telnet 10.10.54.75 : 10.10.54.75 ip 주소로 telnet연결이 된다.
  • df -l -k : 파티션 공간 나뉜 모습 확인 명령

who

who 명령어는 현재 접속한 사용자 정보를 ‘/var/run/utmp’ 파일에서 읽어온다. utmp 파일은 사용자가 원격으로 서버에 로그인할 때 사용자의 정보를 저장하고 사용자가 원격 호스트에서 로그아웃 할 때 저장되어 있는 정보를 제거한다.
현재 서버에 접속해 있는 사용자의 로그인명, 터미널, 로그인 시간, 원격 호스트 또는 X디스플레이를 출력한다.

ls

옵션

옵션 설명
-i inode 번호 표시.
-A, -a 모든 항복을 보여준다(A는 .과 ..은 제외)
-C (column)다중 열 형식의 수직 방식으로 출력을 분류 (ls 명령어의 default)
-s size 각 항목의 크기를 킬로바이트 단위(간접 블록 포함)로 제공합니다.
-F 파일이 디렉토리인 경우 /(슬래시), 파일이 실행 가능한 경우 *(별표), 파일이 소켓인 경우 =(등호), 파일이 선입선출법(FIFO)인 경우 |(파이프), 기호 링크인 경우 @를 각 파일 이름의 뒤에 추가합니다
-d 이름 지정된 디렉토리에 대한 정보만 표시합니다. 디렉토리를 파일처럼 처리 하므로 -l 사용할 때 유용
-R 모든 서브디렉토리를 재귀적으로 나열합니다.
-l 모드, 링크 수, 소유자, 그룹, 크기(바이트), 최종 수정 시간을 표시
-S 파일 크기를 기본으로 내림차순으로 하고 파일 이름을 보조키로 정렬한다.

od 명령

  • 지정된 형식으로 파일을 표시하는 명령.
  • 매개 변수가 없을 땐 표준 입력을 받는다.
  • -t옵션으로 형식 유형을 지정하는데 형식 유형이 지정되지 않으면 -t o2가 기본값이다.(이진수 변환)

옵션

옵션 설명
-t 출력형식 지정 옵션
-x 16진수 표기 보기
-c 아스키 문자로 표시한다.(문자로 표기)

파이프를 이용해 메세지 보내기

  1. who am i 명령 : 로그인 이름, 터미널 이름 및 로그인 시간을 표시
  2. tty 명령 : 현재 터미널 이름을 볼 수 있다.
터미널의 이름을 확인 후 write 명령을 이용해 그곳에 메세지를 보낼 수 있다.
echo HI | write unix104 pts/20
