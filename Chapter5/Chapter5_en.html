What is shell

  • Processes the input command as an intermediate role between the user and the kernel and informs the execution result.
  • Prompt: Displays the status of the system waiting for command input
  • Promport Settings: Look at $PS1.
  • Standard input/output device: Standard input device (keyboard) → shell → monitor (standard output, standard error)
  • File descriptor: Used to change input/output devices: 0,1,2 (standard input, output, error)

Type of Shell

  • Bone Shell (sh)($), Corn Shell (ksh)($), C Shell (csh)(%), Bash Shell ($), etc.
  • You can look at the $SHELL environment variable for the type.
  • Login Shell: Shell automatically created immediately after user logs in
  • Subcells: Shells run by users themselves

Command

  • echo: Output commands that provide both shell commands and utilities
  • print: shell command, more convenient than echo with options

Shell Special Character

*Special characters (meaning all)

  • ls h*: Shows all files starting with h. It also includes h
  • ls h*d: Output all files starting with h and ending with d

? Special Characters

  • Use to represent any one character
  • (The ls hello?.c command looks for helloA.c like this, but does not include hello.c.)(meaning only one character)

[] Special Character

    It represents one of the characters contained in
  • [].
  • ls hello[123].c If you do this, find all hello1,2,3.
  • ls hello[1-3].c Same meaning
  • ls [A-Za-z]*[0-9]: Output all files that start with an English letter and end with a number

Other special characters

  • - and ~: Special characters representing the directory (old directory/home)
  • ; and | (pipes and ;) : A special character that connects commands to commands.
  • " and "" (double quotation marks): String representation, small quotation marks do not interpret, and double quotation marks interpret $'\.
  • 'Special character: Use to receive command execution results as a string
  • ＼(\): Replace special characters with regular characters with reverse slash special characters
  • >><:Change input/output direction (>:Overwrite,>>:Add) (Command 2>File name: Save to standard error file), (Replace command

Variable of Bash Shell

  • Variable: Storage for storing information used by the system or user
  • Shell variable: Variable that is currently available only in the shell (check command set) (variable = value)
  • Environmental variable: Available variable in all shells (check command env) (export variable = value)
  • Variable release unset

Interrupt

  • ctrl + c: Stop current command
  • ctrl + D : eof
  • ctrl + \:quit (current command stop)
  • ctrl + S: stop (screen output stopped)
  • ctrl + Q: start (restart screen output)
  • ctrl +? : erase (delete last character)
  • ctrl + U:kill (delete all command lines)
  • ctrl + W: Delete the last word
  • ctrl + Z: command suspended

Create your own command alias

  • alias alias = 'command'
  • Replace multiple commands with one command
  • Factor fails to hand over factor to required command
  • Release: You can use the alias unalias.

Using Previous Input Commands fc

View previously used commands: fc (fc -l : view by line)

Re-execute command:!

  • Editing a command: arrow key, or Esc + k
  • Exploring commands: Esc + /
  • The last here means the most recent.
Command Discription
!! Last command execution
!n Run the nth command
!-n Run the nth command backwards from the current command
!String Run the last command among commands that begin with the given string
<Event command>:p !n Output to the screen without executing a command designated as an event specifier
^string 1^string 2 Run by replacing string 1 in the last command with string 2

Configuration File

  • Save commands that run automatically each time a user logs in
  • System Initialization File (/etc/profile, /etc/.login)
  • User initialization file ($HOME/).bashrc, $HOME/.profile etc)
  • Run user initialization file: source command
